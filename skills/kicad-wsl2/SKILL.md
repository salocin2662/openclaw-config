# Skill: PCB Layout

Build PCB layouts headlessly via KiCad's `pcbnew` Python API. Places footprints, routes traces, adds copper zones, and writes the board file — all without the KiCad GUI.

---

## What It Does

Takes a netlist from a passing schematic and produces a completed PCB layout: footprint placement following the placement priority hierarchy, trace routing, copper pours, and edge cuts.

Use this skill when Nicolas asks to:
- Build a PCB layout from a schematic
- Place components on a board
- Route a board
- Add copper zones or fill
- Update an existing board layout

---

## Prerequisites

- Schematic must pass ERC (0 errors)
- Netlist must be exported (`.xml` from KiCad)
- All footprints must exist in the project's footprint library or KiCad's built-in libraries
- Design rules must be confirmed with Nicolas before routing starts

---

## How It Works

1. **Confirm**: schematic path, board size, layer stack, design rules
2. **Convert paths**: WSL2 `/mnt/c/` → Windows `C:/` for `pcbnew.LoadBoard()`
3. **Load board**: `pcbnew.LoadBoard(win_path)` or create new from netlist
4. **Place footprints**: follow placement priority (see below)
5. **Route**: apply design rules (trace/space, via size)
6. **Add zones**: copper pours on specified layers
7. **Edge cuts**: define board outline
8. **Save**: `board.Save(win_path)`

---

## Component Placement Priority

**This order is mandatory.** Deviating from it causes routing difficulty.

1. **Mounting holes + tooling fixtures** → establishes board origin and datum
2. **Edge connectors, card-edge connectors, mechanical mounts** → establishes board boundary
3. **Crystals and oscillators** → within 5mm of MCU pins
4. **Decoupling capacitors** → within 2mm of each VDD pin
5. **High-pin-count ICs** (BGA, QFP, TQFP) → plan escape routing before placing these
6. **Medium components, connectors, bulk capacitors**
7. **Passives, fill-in components**

---

## Design Rules (Confirm Before Routing)

| Parameter | Minimum | Preferred |
|-----------|---------|-----------|
| Trace / space | 5 mil | 6–8 mil |
| Via drill | 8 mil | 10 mil |
| Via diameter | 16 mil | 20 mil |
| Annular ring (4-layer) | 5 mil | 6 mil |
| Pad (8 mil drill) | 20 mil | 24 mil |
| Solder mask dam | 4 mil | 5 mil |
| Silkscreen text | 32 mil | 40–50 mil |

**Via types**: through-hole preferred for standard signals; blind/buried require explicit stackup specification.

## Workflows

### Build Layout from Netlist

```
1. Confirm with Nicolas:
   - Schematic/netlist path (Windows path)
   - Board size (e.g. 100mm × 80mm)
   - Layer stack (2-layer or 4-layer, core/prepreg thickness)
   - Impedance requirements (52Ω nominal / 50Ω target if applicable)
   - Design rule file (.kicad_dru) if custom

2. Write build_pcb.py:
   - Convert WSL paths to Windows paths first
   - Import pcbnew: import pcbnew
   - board = pcbnew.CreateNewBoard()
   - board.SetBoardName(project_name)
   - netlist = pcbnew.ImportNetlistFromSchematic(schematic_xml_path)
   - board.BuildBoardFromNetlist(netlist)
   - Place footprints in priority order
   - Route traces: board.Add(track) with pcbnew.F_Cu/B_Cu
   - Add copper zones: board.Add(zone) with appropriate net
   - Draw edge cuts: board.Add(edge)
   - board.Save(win_output_path)

3. Execute:
   "C:\Program Files\KiCad\10.0\bin\python.exe" build_pcb.py

4. Present placement summary:
   - Footprint count by category (IC, passives, connectors)
   - Board outline dimensions
   - Layer stackup
   - Trace count (by net)
   - Via count

5. Wait for Nicolas approval, then finalize routing and zones
```

### Update Existing Board

```
1. Confirm board file path and schematic path
2. Run kicad-inspect skill to see current state
3. Convert paths: ws_path.replace('/mnt/c/', 'C:/')
4. Load board: pcbnew.LoadBoard(win_path)
5. Check for netlist changes since last layout
6. Update/add/remove footprints as needed
7. Re-route affected traces
8. Save: board.Save(win_path)
```

## Notes

- **Place before route**: always complete footprint placement before starting trace routing
- **Decoupling caps first**: within 2mm of every VDD pin — this is non-negotiable for production boards
- **BGA escape routing**: plan fan-out before placing BGA packages — failure to do this makes routing impossible
- **Via size minimum**: 8 mil drill / 16 mil diameter. Preferred: 10 mil drill / 20 mil diameter
- **Annular ring**: 8 mil drill → minimum 20 mil pad (6 mil ring). Use 24 mil pad for production
- **KiCad version awareness**: Windows runs KiCad 10.0.4, WSL2 runs KiCad 9.0.8 (headless). The Python bindings (`pcbnew`) work on both. Do NOT assume a single version — confirm with Nicolas before running any script.
