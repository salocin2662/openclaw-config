# Skill: Schematic Generation

Generate KiCad schematics via `kicad-sch-api`. Creates component symbols, wires, nets, power rails, and saves as `.kicad_sch` format (KiCad 10).

---

## What It Does

Uses `kicad_sch_api` (the `kicad-sch-api` Python package) to programmatically build a KiCad schematic from a component list. Creates symbols, places them on the sheet, wires nets, adds power flags, and runs ERC to verify electrical correctness.

Use this skill when Nicolas asks to:
- Generate a new schematic from a bill of components
- Add components to an existing schematic
- Wire up a netlist
- Check ERC results

---

## How It Works

1. **Confirm project details**: name, path, sheet size
2. **Parse component list**: reference designators, values, packages, symbols
3. **Execute build script**: `kicad-sch-api` via `python.exe`
4. **Run ERC**: validate electrical rules, confirm 0 errors
5. **Report**: netlist summary, component count, ERC status

---

## Workflows

### Generate New Schematic

```
1. Confirm with Nicolas:
   - Project name and path (Windows path, e.g. C:/Users/<username>/Desktop/project/)
   - Sheet size (A4, A3, A2, etc.)
   - Board description and netlist

2. Write build_schematic.py:
   - Import ksa from kicad_sch_api
   - ksa.create_schematic(project_path, sheet_size)
   - For each component: sch.components.add(ref, value, package, symbol_lib)
   - For each net: sch.wires.add_net(net_name, [list of connected refs/pins])
   - Add power flags (GND, VCC rails)
   - sch.save(output_path)

3. Execute:
   "C:\Program Files\KiCad\10.0\bin\python.exe" build_schematic.py

4. Run ERC:
   kicad-cli sch run erc <project>.kicad_sch
   Or via kicad-sch-api: ksa.run_erc(schematic_path)

5. Parse ERC output:
   - 0 errors: ERC pass → report netlist summary
   - Errors: list each error with sheet, net, and pin
   - Wait for Nicolas to fix schematic, then re-run

6. Save netlist:
   - Export .xml netlist for use in pcb-layout skill
   - Report: component count, net count, power rails, sheet count
```

### Add Components to Existing Schematic

```
1. Confirm schematic path and current ERC status
2. List new components to add (ref, value, package)
3. Write incremental build script:
   - Load existing schematic: ksa.load_schematic(existing_path)
   - sch.components.add() for each new component
   - sch.wires.add_net() for any new nets
   - ksa.save_schematic(updated_path)
4. Execute and run ERC
5. Report updated component/net count
```

## Script Templates

### build_schematic.py

```python
#!/usr/bin/env python3
"""
build_schematic.py — Generate KiCad schematic for <project_name>
Generated by: nic-pcb-agent schematic-gen skill
"""

import sys
import os

# kicad-sch-api
sys.path.insert(0, "C:/Users/<username>/AppData/Roaming/Python/Python311/site-packages")
from kicad_sch_api import ksa

PROJECT_NAME = "<project_name>"
PROJECT_PATH = r"C:\Users\<username>\Desktop\<project_name>"
OUTPUT_PATH = os.path.join(PROJECT_PATH, f"{PROJECT_NAME}.kicad_sch")

# Components: (ref, value, package, symbol)
COMPONENTS = [
    ("U1", "STM32F405", "QFP-64", "MCU_ST_STM32F4"),
    ("C1", "100nF", "0402", "Device:C"),
    ("R1", "10k", "0402", "Device:R"),
    # Add components here
]

# Nets: (net_name, [list of component/pin connections])
# Format: "REF:PIN" or "net_label"
NETS = [
    ("VCC", ["U1:VDD", "U1:VDDA", "C1:1", "R1:1"]),
    ("GND", ["U1:GND", "C1:2", "R1:2"]),
    # Add nets here
]

def main():
    print(f"Creating schematic: {PROJECT_NAME}")
    sch = ksa.create_schematic(PROJECT_PATH, sheet_size="A4")
    sch.set_title(PROJECT_NAME)
    sch.set_revision("A")
    sch.set_company("")

    # Add components
    for ref, value, package, symbol in COMPONENTS:
        sch.components.add(ref=ref, value=value, package=package, symbol=symbol)
        print(f"  Added: {ref} = {value} ({package})")

    # Add nets
    for net_name, connections in NETS:
        sch.wires.add_net(net_name, connections)
        print(f"  Net: {net_name} -> {connections}")

    # Add power flags
    sch.wires.add_power_flag("GND", "PWR_FLAG:GND")
    sch.wires.add_power_flag("VCC", "PWR_FLAG:VCC")

    # Save
    sch.save(OUTPUT_PATH)
    print(f"\nSchematic saved: {OUTPUT_PATH}")

    # Run ERC
    erc_result = ksa.run_erc(OUTPUT_PATH)
    if erc_result.errors == 0:
        print(f"ERC: PASS — {erc_result.warnings} warnings, {erc_result.errors} errors")
    else:
        print(f"ERC: FAIL — {erc_result.warnings} warnings, {erc_result.errors} errors")
        for err in erc_result.error_list:
            print(f"  [{err.sheet}] {err.net}: {err.message}")

if __name__ == "__main__":
    main()
```

## Quick Commands

| Command | Action |
|---------|--------|
| `generate schematic <project>` | Generate full schematic for named project |
| `add component <ref> to schematic` | Add one component to existing schematic |
| `run ERC on <schematic>` | Run electrical rule check and report |
| `netlist for <project>` | Export netlist XML from schematic |

---

## Notes

- **ERC is mandatory**: never report a schematic as "ready for layout" without passing ERC (0 errors)
- **Symbol library**: components must exist in the project's symbol library or KiCad's native libraries. If a symbol is missing, report the missing symbol and wait for Nicolas to add it
- **Power rails**: always add power flags to VCC and GND nets — ERC requires them
- **Path convention**: Windows KiCad Python uses Windows paths. WSL2 users must convert `/mnt/c/` → `C:/`
